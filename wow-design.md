# Tradie Front Desk Wow Design System

This file is the visual source of truth for the current v2 site direction. The style is intentionally loud, local, urgent, and built to stop the scroll for trade business owners.

## Brand Direction

Tradie Front Desk should feel like a comic-book local lead-generation poster rather than a quiet SaaS website.

The brand feeling is:

- Loud
- Confident
- Local
- Energetic
- Simple
- Opportunity-driven
- Urgent

The site should feel like:

- A superhero comic cover
- A local service flyer
- A high-converting Facebook ad
- A wanted poster for trade businesses
- A bright cartoon mascot brand
- A practical demand-capture offer

It should not feel:

- Corporate
- Premium-minimal
- Beige
- Soft SaaS
- Generic AI dashboard
- Robotic
- Overly polished agency brochure

## Core Message

The strongest public message is:

> Good jobs are calling. Stop letting them slip away.

Support this with:

- Missed calls
- Slow replies
- Quote follow-up
- Owner handoff
- Clear front-desk process
- More organised opportunities
- Less manual chasing

Avoid leading with:

- CRM
- AI
- Automation
- Funnels
- SaaS
- Workflows
- GoHighLevel
- Bots

The technology is there, but the buyer-facing story is practical front-desk job handling.

## Safe Claims

Keep the claims strong but legally safe.

Allowed:

- Helps catch missed calls
- Helps reply faster
- Helps follow up quotes
- Helps keep opportunities visible
- Helps organise enquiries
- Helps move customers toward callbacks, quote requests, site visits, or bookings
- Designed to reduce missed opportunities

Avoid:

- Guaranteed jobs
- Guaranteed leads
- Guaranteed revenue
- Guaranteed profit
- Guaranteed rankings
- Guaranteed reviews
- Guaranteed bookings
- Exact ROI claims
- Exact customer behaviour claims

Preferred CTA language:

- Book Free Audit
- Claim My Free Audit
- Find My Enquiry Leaks
- See The System
- View Packages

## Typography

Typography is the main part of the brand.

Main headline direction:

- Big
- Heavy
- Comic style
- All caps feel
- Thick black outlines
- Strong shadows
- Slightly angled or poster-like
- Built to read instantly

Current font stack:

- Display: `Bangers`, `Arial Black`, `Impact`, sans-serif
- Heavy UI: `Anton`, `Impact`, `Arial Black`, sans-serif
- Body: `Inter`, system sans-serif

Use display fonts for:

- Hero headlines
- Section headlines
- Major visual labels
- Poster-style emphasis

Use heavy UI fonts for:

- Navigation
- Buttons
- Eyebrows
- Pills
- Table headers
- Form labels
- Pricing labels

Use body font for:

- Paragraphs
- Legal text
- Supporting copy
- FAQ answers

## Headline Treatment

Hero headlines use white comic text with black outline and pink drop shadow.

The current style is:

- White fill
- Multiple black text shadows to fake a thick outline
- Hot pink offset shadow
- Very large type
- Tight line height

Headlines should feel like:

- `PAINTER WANTED!`
- `GOOD JOBS ARE CALLING`
- `STOP LETTING THEM SLIP AWAY`
- `URGENT PLUMBING CALLS NEED A FAST FRONT DESK`

Avoid long thin headlines. If a line gets too long, rewrite the copy rather than shrinking the brand.

## Layout Rules

The page uses a loud poster structure.

Default layout elements:

- Yellow body background
- Electric blue hero panels
- Diagonal yellow/blue split blocks
- Halftone dot texture
- Thick black borders
- Hard offset shadows
- No rounded soft SaaS cards
- No beige or premium muted surfaces

Cards:

- Border: 4-5px black
- Shadow: hard black offset
- Background: white, yellow, pale pink, or pale blue
- Shape: square corners
- Texture: subtle halftone/dot overlay

Buttons:

- Border: 4px black
- Square corners
- Uppercase heavy type
- Hard black shadow
- Red primary CTA
- Yellow or blue secondary CTA

Sections:

- Full-width bands are preferred
- Use thick black separators
- Use poster-like colour blocks
- Avoid nested card-in-card layouts

## Navigation

The navbar should be simple and punchy.

Current direction:

- Yellow background
- Halftone dots
- Thick black bottom border
- Logo in a small bordered square
- Brand text: `Tradie Front Desk`
- CTA: red `Book Free Audit`

Navbar copy should stay short.

Preferred nav labels:

- Home
- The System
- Pricing
- Proof
- Claim Audit

## Hero Pattern

The hero should be a first-viewport poster.

Required elements:

- Loud eyebrow badge
- Massive comic headline
- White bordered text panel
- Red primary CTA
- Yellow secondary CTA
- A square Remotion animation panel
- Optional `WANTED` stamp
- Diagonal yellow/blue background

Hero should not be a calm split-card SaaS layout. It should feel like an ad creative that happens to be a website.

## Remotion Video Direction

Videos should match the site.

Use:

- 1:1 square format
- Black canvas
- Yellow, blue, pink, red accents
- Big comic text
- Thick borders
- Halftone/grid texture
- Animated SVG lines
- Moving cards and labels
- Simple words that explain without audio

Avoid:

- Robot imagery
- Stock footage
- Generic AI dashboards
- Tiny unreadable UI
- Beige/coral legacy palette
- Audio for now

The videos should explain visually:

- Missed call caught
- Fast reply sent
- Job details collected
- Owner alerted
- Quote followed up
- Opportunity kept visible

## Page Tone

The writing should be practical and urgent, not technical.

Good phrases:

- Good jobs are calling.
- Stop letting them slip away.
- Catch missed calls before they go cold.
- Reply before the customer calls the next tradie.
- Keep quote requests visible.
- Follow up without relying on memory.
- One front-desk process for every opportunity.

Weak phrases:

- Transform your operations.
- Seamless AI automation.
- End-to-end CRM workflow.
- Optimize your funnel.
- Unlock scalable revenue.
- Guaranteed booked jobs.

## Page-Specific Direction

Home:

- Lead with missed opportunities and demand capture.
- The page should feel like the main poster.
- Keep the hero loud and simple.

How It Works:

- Explain the five-step flow.
- Make each step feel like a comic panel.
- Keep the language clear: Capture, Respond, Qualify, Book, Follow Up.

Pricing:

- Keep it bold but credible.
- Make the flagship package feel visually dominant.
- Avoid hiding starting prices.
- Make clear that final pricing depends on scope.

Proof:

- Show system activity, not risky outcome claims.
- Focus on calls caught, replies sent, follow-ups completed, bookings created, and reviews requested.

Free Audit:

- Make it feel like an immediate opportunity check.
- The audit should feel easy, fast, and practical.

Trade Pages:

- Use trade-specific urgent language.
- Electricians: faults, switchboards, EV chargers, urgent calls.
- Plumbers/drainage: leaks, blocked drains, after-hours triage.
- Roofers: inspections, repairs, high-value quotes, weather delays.

Legal Pages:

- They can use the same brand shell, but body content must remain readable.
- Do not use animated hero text for long legal titles.

## Accessibility And Responsiveness

Because the design is loud, readability must be checked carefully.

Rules:

- Text must not overflow on mobile.
- Hero headline can wrap aggressively, but must stay readable.
- Buttons must fit their text.
- Remotion video panels must not cover CTAs.
- Avoid tiny text inside animations where possible.
- Legal and FAQ content must remain readable on white cards.

Mobile:

- Stack hero content.
- Keep CTA buttons full width.
- Keep logo and nav CTA visible.
- Hide desktop nav links when needed.
- Avoid huge off-screen decorative text.

## QA Checklist

Before finalizing major design changes:

- Run `npm run build`
- Check the homepage desktop
- Check the homepage mobile
- Check pricing desktop
- Check how-it-works mobile
- Check legal pages for readability
- Confirm no text is cut off
- Confirm the site does not run on port `4321` if another project is using it

Preferred local server for this project:

```bash
npm run dev -- --host 127.0.0.1 --port 4322
```
