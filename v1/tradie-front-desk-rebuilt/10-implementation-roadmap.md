# Implementation Roadmap

## Goal

Launch Tradie Front Desk with a clear business model, practical offer, and conversion-focused website that trade business owners can understand without technical knowledge.

The launch priority is not to show every GoHighLevel or AI feature. The priority is to make the buyer believe:

> This helps me stop losing jobs from missed calls, slow replies, and forgotten follow-up.

## Phase 1: Lock The Offer

### Outcomes

- final category
- final core promise
- final product ladder
- final pricing ranges
- final audit CTA
- final guarantee

### Decisions To Use

Category:

> Managed front-desk system for tradies.

Core promise:

> More booked jobs. Less chasing. No tech headaches.

Hero angle:

> Stop losing good jobs from missed calls, slow replies, and forgotten follow-up.

Product ladder:

- Front Desk Starter
- Booked Jobs System
- Managed Growth System

Primary CTA:

> Book a Free Front Desk Audit

Guarantee:

> The agreed system gets built, tested, launched, and fixed if a proposal-listed part is not working.

### Completion Criteria

- One-page offer summary exists.
- Package cards are final enough for website use.
- Pricing ranges are approved.
- Add-ons are clearly secondary.
- AI is positioned as optional and controlled.

## Phase 2: Build The Audit Asset

The Free Front Desk Audit is the main conversion asset.

### Create

- audit booking page
- audit questionnaire
- Front Desk Scorecard
- audit call structure
- follow-up recommendation template

### Audit Promise

Use this promise on the audit page:

> In around 20 minutes, we show where enquiries are leaking, what it may be costing you, and which setup would fix it.

The audit should clearly state:

- no obligation
- no lead pitch
- no prep required beyond knowing how enquiries currently arrive
- buyer leaves with scorecard, leak list, priority fixes, and package recommendation

### Scorecard Categories

- missed-call handling
- response speed
- website enquiry capture
- quote request clarity
- booking/callback path
- quote follow-up
- old lead database
- social message handling
- Google message handling
- review request process
- pipeline visibility
- campaign speed-to-lead

### Completion Criteria

- a visitor can book the audit online
- confirmation and reminders are sent
- audit questions identify package fit
- audit output recommends clear next steps
- form uses only essential first-step fields: name, business, trade, location, website, phone, email, biggest issue, preferred time

## Phase 3: Build Proof Assets

- sample missed-call flow

For each client, collect:

- enquiries captured
- missed calls recovered
- instant replies sent
- follow-ups sent
- replies received
- callbacks booked
- quote/site visits booked
- old leads contacted
- reviews requested
- reviews gained

### Completion Criteria

- at least one proof block can be used on the homepage
- proof page has at least demo assets if live case studies are not ready
- claims are modest and evidence-based

## Phase 4: Website Build

### Recommended Launch Pages

Build these first:

1. Home
2. How It Works
3. What Is Included
4. Free Front Desk Audit
5. Pricing
6. Proof / Case Studies
7. Contact
8. Privacy Policy
9. Terms

### Trade Landing Pages

Build these early if outreach or ads target specific trades:

- Electricians
- Plumbers / Drainage
- Roofers

Each trade page must include:

- trade-specific pain points
- trade-specific enquiry examples
- trade-specific proof or sample screenshot
- trade-specific CTA feeding the same audit flow
- compatibility/owner-control objection handling

### Website Requirements

Each core page must include:

- clear headline
- plain-English copy
- one main CTA
- proof or example wherever the page makes a conversion claim
- no technical jargon in hero sections
- audit CTA near top and bottom

### Completion Criteria

- homepage explains offer in under 30 seconds
- audit CTA is visible above the fold
- pricing has starting ranges or clear package logic
- proof page exists even if early proof is demo/anonymised
- mobile booking flow is simple

## Phase 6: Sales System

### Create

- audit call script
- package recommendation script
- pricing explanation script
- objection handling sheet
- proposal template
- follow-up sequence
- lost deal follow-up

### Sales Process

1. Audit booked.
2. Audit completed.
3. Leaks identified.
4. Package recommended.
5. Proposal sent.
6. Follow-up sent.
7. Client onboarded.
8. System built.
9. System launched.
10. Monthly report delivered.

### Completion Criteria

- every audit has a follow-up path
- proposals use the same package structure as the website
- guarantee and exclusions are clear
- no one sells guaranteed job numbers

## Phase 7: Launch Campaign

### Initial Outreach Angles

Use:

- missed-call recovery
- quote follow-up
- old lead reactivation
- website form leak
- anti-lead-seller
- review growth

### Target First

Start with:

- electricians
- plumbers / drainage
- roofers

Why:

- strong missed-call pain
- quote/site-visit process
- urgency
- follow-up value

### Completion Criteria

- at least three outreach message angles are live
- booking page is connected
- audit reminders work
- follow-up sequence works
- responses are tracked by source

## Phase 8: Monthly Delivery And Reporting

### Monthly Client Report

Include:

- enquiries captured
- missed-call replies
- response activity
- follow-ups sent
- replies received
- callbacks/bookings
- reviews requested
- old leads contacted
- notable opportunities
- next improvements

### Monthly Optimisation

Review:

- where customers drop off
- which follow-ups get replies
- missed-call volume
- quote follow-up performance
- reactivation response
- campaign lead speed
- owner handoff issues

### Completion Criteria

- clients see practical activity, not technical reports
- reports connect to booked opportunities
- next actions are clear

## Acceptance Criteria

The launch system is ready when:

- a tradie can understand the offer without hearing "CRM" or "AI" first
- the website clearly explains the missed-call and follow-up problem
- the audit can be booked without manual back-and-forth
- the sales call can diagnose leaks and recommend a package
- the package offer does not feel like a random menu of features
- the proof system is honest and practical
- the delivery system can be repeated for multiple clients

## Final Rule

Do not let the business become a menu of AI and automation features.

Everything should ladder back to:

> Capture enquiries, reply fast, follow up, and book more jobs.
